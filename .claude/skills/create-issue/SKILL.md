# create-issue スキル

`kawafuchieirin/regular-study` リポジトリ専用の Issue 作成スキル。
勉強会テーマ・機能追加・バグ報告など、このリポの慣習に沿った Issue を作成する。

## 前提

- `gh` CLI がインストール・認証済みであること
- 対象リポジトリは固定: `kawafuchieirin/regular-study`
- リポジトリ運用の慣習（このスキルを使う上で必ず守る）:
  - 勉強会テーマの Issue はタイトルを「勉強会: <内容>」または「<内容> 勉強会」形式にする
  - 機能追加・改善は `enhancement` ラベルを付ける
  - 勉強会テーマ Issue は README.md の `## 勉強会内容` 表にも追記する

## 実行フロー

以下のステップを **順番に** 実行する。

### Step 1: タイトル決定

- `$ARGUMENTS` が渡されていればそれをタイトル候補として使う
- 渡されていなければユーザーにタイトルを質問する
- タイトルが曖昧な場合（短すぎ・抽象的すぎ）は、より具体的にすることを提案する

### Step 2: イシュー種別の判定とラベル決定

タイトルと文脈から種別を推定し、ユーザーに確認する。種別ごとのラベル:

| 種別 | 判定の目安 | 付与ラベル |
|---|---|---|
| 勉強会テーマ | タイトルに「勉強会」「会:」を含む | （ラベル無し） |
| 機能追加・改善 | 「追加」「対応」「改善」「導入」等を含む | `enhancement` |
| バグ | 「修正」「不具合」「動かない」等を含む | `bug`（無ければ作成せず確認） |
| その他 | 上記以外 | （ラベル無し） |

判定に迷う場合は必ずユーザーに質問する。勝手に判断しない。

### Step 3: 本文の作成

ユーザーに以下を質問して本文を組み立てる。簡潔で構わない（既存 Issue は本文が短いものが多い）:

- 概要（何をやるか・何が問題か）を1〜3行で
- 参考資料の URL があれば
- 受け入れ条件・完了の定義があれば

ユーザーが「特に無し」と答えた項目はスキップする。本文が空でも作成可（既存 Issue にも空のものがある）。

### Step 4: 内容の確認

組み立てた以下の情報をユーザーに提示し、作成して良いか確認する:

```
タイトル: <title>
ラベル:   <label or なし>
本文:
<body or "(空)">
```

ユーザーから OK が出るまで修正を受け付ける。

### Step 5: Issue 作成

`gh` コマンドで作成する。本文は HEREDOC で渡す（特殊文字対策）:

```bash
gh issue create \
  --repo kawafuchieirin/regular-study \
  --title "<title>" \
  --body "$(cat <<'EOF'
<body>
EOF
)" \
  --label "<label>"   # ラベル無しの場合は --label 行ごと省略
```

作成成功時に表示される Issue URL とイシュー番号をユーザーに報告する。

### Step 6: 勉強会テーマなら README.md を更新

Step 2 で「勉強会テーマ」と判定された場合のみ実行する。それ以外はスキップして終了。

1. `README.md` の `## 勉強会内容` 表（`|回数|内容|` で始まる）を Read で読む
2. 既存の最終行の回数 + 1 を新しい回数とする
3. 以下の形式で行を追加する:

   ```
   |<新回数>|[<タイトル本体>](https://github.com/kawafuchieirin/regular-study/issues/<新issue番号>)|
   ```

   - 「タイトル本体」はイシュータイトルから「勉強会: 」「勉強会:」のプレフィックスを除いたもの
   - 例: `勉強会: Terraform入門` → `[Terraform入門](.../issues/N)`

4. Edit ツールで `README.md` を更新する
5. ユーザーに「README.md を更新しました。コミット・プッシュしますか?」と確認する
6. OK なら以下でコミット（`git push` はユーザー確認後のみ）:

   ```bash
   git add README.md
   git commit -m "docs: 勉強会テーマ #<新issue番号> を README に追記"
   ```

## 注意事項

- ユーザー確認なしに `gh issue create` を実行してはならない
- リポジトリは常に `kawafuchieirin/regular-study` 固定。他のリポジトリには作成しない
- `gh auth status` でエラーが出た場合は `gh auth login` を案内して終了
- 本文に個人情報・クレデンシャル（トークン・パスワード等）が含まれていないか軽くチェックする（パブリックリポジトリのため）
