# スクール成果アウトプット作成

## 概要
勉強会やスクールで学んだ内容を、定型フォーマットに沿って構造化されたマークダウンとして出力する。

## 手順

### 1. 情報収集
- 引数 `$ARGUMENTS` が与えられた場合はそれをベースにする
- 引数がない・情報が不足している場合は、以下の5項目についてユーザーに確認する
- 現在のリポジトリやブランチの内容から、成果物の情報を推測できる場合は積極的に活用する

### 2. 必要な情報（5項目）

ユーザーから以下の情報を収集する。不明な項目はユーザーに質問して埋める。

| # | 項目 | 説明 | 例 |
|---|------|------|----|
| ① | 成果の内容 | 何を目的に、何ができるものを作ったか | 〜を目的に、〜ができる〜を作成しました |
| ② | 成果物のURL/見てわかるもの | Qiita記事、Webアプリ、スクリーンショット等 | QiitaのURL、WebアプリのURL、スクショ |
| ③ | スクール受講開始前の状態 | 受講前のスキルレベル | Python入門を少しだけ見たことがある程度 |
| ④ | 期間と時間 | スクール受講からどのくらいの期間・時間でできたか | スクール受講開始から3ヶ月で合計70時間 |
| ⑤ | スクールを通して学べたこと | 成果物作成にあたりスクールで学んだスキル・知識 | 副業獲得方法全般、エラーの対処法 |

### 3. マークダウン出力
- 作業ディレクトリに `output.md` を作成（ファイル名の指定があればそれに従う）
- 下記テンプレートに沿って出力する

### 4. 確認
- 出力内容をユーザーに提示し、修正があれば対応する
- 最終的なファイルパスを報告する

## テンプレート

```markdown
# 成果発表

## ① 成果の内容
[成果の説明を記載]

## ② 成果物のURL / 見てわかるもの
[URL やスクリーンショットへのリンクを記載]

## ③ スクール受講開始前の状態
[受講前のスキルレベルや経験を記載]

## ④ スクール受講から何時間でどのくらいの期間でできたのか
[期間と時間を記載]

## ⑤ 成果物の作成にあたりスクールを通して学べたこと
[学んだスキルや知識を記載]
```

## 注意事項
- 各項目はユーザーの言葉をできるだけ活かし、自然な文章に整える
- 情報が不足している場合は推測せず、必ずユーザーに確認する
- 出力ファイルのパスは最後に必ず報告する
