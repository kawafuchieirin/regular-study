# Output スキル

`output/` 配下の Markdown ファイルを読み込み、スクール成果フォーマット（5項目）の文章を AI が生成し、ユーザー確認後に Slack へ投稿するスキルです。

## 実行フロー

以下のステップを **順番に** 実行してください。

### Step 1: ファイル読み込み

- `output/$ARGUMENTS` を Read ツールで読み込む
- ファイルが存在しない場合は、`output/` ディレクトリ内の `.md` ファイル一覧を Glob で取得し、ユーザーに選択肢として提示して選んでもらう
- `_slack.txt` で終わるファイルと `profile.md` は一覧から除外する
- **プロフィール読み込み**: `output/profile.md` を Read ツールで読み込む。このファイルには③④の固定値が記載されている。存在しない場合はユーザーに質問して補完する

### Step 2: 5項目テンプレートに沿った文章生成

読み込んだファイルの内容を分析し、以下の5項目テンプレートに沿った文章を **AIが生成** する。

```
①成果の内容
〜を目的に、〜ができる〜を作成しました。

②成果物のURL/見てわかるもの
URL: 〜
スクリーンショット: 〜

③スクール受講開始前の状態
〜

④スクール受講から何時間でどのくらいの期間でできたのか
スクール受講開始から〜ヶ月で合計〜時間

⑤成果物の作成にあたりスクールを通して学べたこと
〜
```

### Step 3: 不足情報の補完

ファイルの内容だけでは判断できない情報について、ユーザーに質問して補完する。

- **③と④は `output/profile.md` から自動で埋める**（ユーザーへの質問は不要）
- `profile.md` が存在しない場合のみ、③④もユーザーに質問する

よくある不足情報の例（質問が必要なもの）:

- 成果物の URL やスクリーンショット
- その他、ファイルから読み取れなかった項目

質問は一度にまとめて行い、ユーザーの負担を減らすこと。

### Step 4: 文章の提示と確認

- 生成した5項目の文章をユーザーに提示する
- 修正が必要かどうか確認する
- 修正要望があれば反映して再提示する
- ユーザーから OK をもらうまで繰り返す

### Step 5: Slack mrkdwn 変換と保存

ユーザーから OK をもらったら、以下の変換ルールで Slack mrkdwn 形式に変換する:

| Markdown | Slack mrkdwn |
|---|---|
| `# 見出し` / `## 見出し` / `### 見出し` | `*見出し*` |
| `**太字**` | `*太字*` |
| `[テキスト](URL)` | `<URL\|テキスト>` |
| `- リスト項目` | `• リスト項目` |

変換後のテキストを `output/元ファイル名_slack.txt` に保存する（例: `test.md` → `output/test_slack.txt`）。

### Step 6: Slack 投稿

- 投稿前に必ずユーザーに「Slack に投稿してよいですか？」と確認する
- 環境変数 `SLACK_USER_TOKEN` と `SLACK_CHANNEL_ID` を使用する
- 以下の curl コマンドで投稿する:

```bash
curl -s -X POST https://slack.com/api/chat.postMessage \
  -H "Authorization: Bearer $SLACK_USER_TOKEN" \
  -H "Content-Type: application/json; charset=utf-8" \
  -d "$(jq -n --arg channel "$SLACK_CHANNEL_ID" --arg text "$(cat output/元ファイル名_slack.txt)" '{channel: $channel, text: $text, unfurl_links: false}')"
```

- 投稿結果の `ok` フィールドを確認し、成功/失敗をユーザーに報告する
- 環境変数が未設定の場合はエラーメッセージを表示し、設定方法を案内する

## 注意事項

- 文章生成では、ファイルの内容を尊重しつつ、テンプレートの形式に合うように適切に要約・整形する
- ユーザーの確認なしに Slack 投稿を実行してはならない
- 5項目すべてが埋まっている状態で投稿する（不明な項目はユーザーに確認する）
