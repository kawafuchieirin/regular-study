# 成果物まとめ

## ①成果の内容

社内ドキュメントや仕様書のような複数ファイルを対象に、AIエージェントがファイルシステムを直接探索しながら回答できる Q&A ボットを作成しました。

この成果物では、ベクトル DB を使わずに S3 Files に配置したドキュメントを Lambda 上へマウントし、Strands Agents と Amazon Bedrock を使って必要なファイルを検索・読取・要約しながら回答できる構成を実装しました。

あわせて、以下のような機能を一通り実装しました。

- React によるチャット UI
- FastAPI によるチャット API
- 会話履歴を保存する DynamoDB 連携
- ドキュメント探索用のカスタムツール実装
- Terraform による AWS インフラ構築
- CloudFront + S3 によるフロントエンド配信
- 設計図、構成図、データフロー図などのドキュメント整備

## ⑤成果物の作成にあたりスクールを通して学べたこと

この成果物の作成を通して、以下のことを学べました。

- 要件をもとに画面・API・インフラまで含めて全体設計する進め方
- React と FastAPI を連携させてアプリケーションを組み立てる方法
- DynamoDB を使った会話履歴管理の考え方
- AWS Lambda、S3、CloudFront、Terraform を使ったデプロイ構成の作り方
- 生成 AI をアプリケーションに組み込むときに、プロンプトだけでなくツール設計やデータの持たせ方が重要になること
- 図やドキュメントを整備しながら、第三者に伝わる形で成果物をまとめることの大切さ

特に、今回の成果物では「AI に答えさせる」だけではなく、「どのファイルを探し、どの情報を根拠に回答するか」を考えながらツールを設計した点が大きな学びでした。
