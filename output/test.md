# SKILL.md 改修内容

## 概要

SKILL.md の改修を実施し、引数処理・出力管理・Slack連携の各フローを整理しました。

---

## 主な変更点

### 1. 引数の分岐ロジック

引数の内容に応じて処理を分岐するように変更しました。

* **パターンA（ファイル指定）**

  * `.md` で終わる引数を受け取った場合に適用
  * 指定されたMarkdownファイルを入力として処理を行う

* **パターンB（従来フロー）**

  * `.md` 以外の引数の場合に適用
  * 従来通り、既存仕様に基づく5項目を収集して処理を行う

---

### 2. 保存先の変更

出力ファイルの保存先を以下のように変更しました。

* **変更前**

  ```
  /output.md
  ```

* **変更後**

  ```
  /output/XXXX.md
  ```

* ディレクトリ単位での管理に変更

* ファイル名（XXXX）はユニークに生成される

---

### 3. Slack mrkdwn 変換ルールの追加

Slack投稿用フォーマット（mrkdwn）への変換処理を追加しました。

#### 変換ルール

* 見出しの変換
  `### 見出し` の形式を、Slack上で適切に表示される形式へ変換する

* リストの変換
  `- リスト項目` を `• リスト項目` に変換する

---

### 4. フロー整理

処理フローをパターンごとに明確に分離しました。

#### パターンA（ファイル入力）

1. Markdownファイルを読み込む
2. 内容を整形する
3. Slack投稿用フォーマットへ変換する
4. 出力ファイルを生成する

#### パターンB（従来入力）

1. 必要な5項目を収集する
2. 内容を整形する
3. Slack投稿用フォーマットへ変換する
4. 出力ファイルを生成する

---


---

## まとめ

* 引数による処理分岐を導入し、柔軟性を向上させた
* 出力ファイルをディレクトリ管理へ変更し、管理性を改善した
* Slack向けフォーマット変換ルールを明確化した
* フローを整理し、可読性と保守性を向上させた
