# output ディレクトリについて

このディレクトリは、勉強会や学習の中で作成したアウトプットをまとめて置くための場所です。

作成したプロフィール文、提出用の文章、学習メモなどを用途ごとに保存していく想定です。

また、`.claude/skills/output/SKILL.md` で定義されている `output` スキルが参照するディレクトリでもあります。

## 含まれているファイル

| ファイル名 | 内容 |
| --- | --- |
| `profile.md` | スクール成果プロフィールの固定値メモ |

## skill での使われ方

`output` スキルでは、このディレクトリ内の Markdown ファイルを読み込み、スクール成果フォーマットの文章を生成します。

主な流れは次のとおりです。

1. `output/` 配下の対象 Markdown ファイルを読み込む
2. `output/profile.md` を参照して、プロフィールの固定値を補完する
3. 5項目形式の成果文章を生成する
4. ユーザー確認後に Slack 投稿用テキストを作成する
5. 必要に応じて `*_slack.txt` を生成し、Slack へ投稿する

## 使い方

- 学習の成果物を Markdown 形式で保存する
- 後から見返しやすいように、ファイル名は内容がわかる名前にする
- 公開リポジトリのため、個人情報や機密情報は含めない

## skill を使うための書き方

- 成果物ごとに `output/` 配下へ `.md` ファイルを作成する
- 成果の内容、URL、学んだことなど、文章生成に必要な情報をできるだけ書いておく
- ③と④の固定情報は `output/profile.md` にまとめて管理する
- Slack 投稿用に自動生成される `*_slack.txt` は中間成果物として扱う

例:

```text
output/
├── README.md
├── profile.md
├── app-review.md
└── app-review_slack.txt
```

### Markdown ファイルに書く内容の例

```md
# アプリ制作のアウトプット

## 成果の概要
家計簿アプリを作成しました。

## URL
https://example.com

## 工夫したこと
- ログイン機能を実装した
- 月別の集計を表示できるようにした

## 学べたこと
React の状態管理と API 連携の流れを学べました。
```

### skill 利用時の想定

- `output` スキルに対象ファイル名を渡して実行する
- 例: `app-review.md` をもとに成果文章を生成する
- 不足情報があれば追加確認される
- 確認後、`output/app-review_slack.txt` のような Slack 投稿用ファイルが作成される
