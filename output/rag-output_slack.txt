*①成果の内容*

社内ドキュメントや仕様書のような複数ファイルを対象に、ベクトルDBを使わずに AI エージェントがファイルシステムを直接探索して回答できる Q&A ボットを作成しました。S3 Files に配置したドキュメントを Lambda 上にマウントし、Strands Agents と Amazon Bedrock を組み合わせて、必要なファイルを検索・読取・要約しながら回答する構成です。React 製チャット UI、FastAPI 製チャット API、DynamoDB による会話履歴保存、ドキュメント探索用カスタムツール、Terraform による AWS インフラ構築、CloudFront + S3 によるフロントエンド配信まで、一通り実装しました。

*⑤成果物の作成にあたりスクールを通して学べたこと*
この成果物の作成を通して、以下のことを学べました。
• 要件をもとに画面・API・インフラまで含めて全体設計する進め方
• React と FastAPI を連携させてアプリケーションを組み立てる方法
• DynamoDB を使った会話履歴管理の考え方
• AWS Lambda、S3、CloudFront、Terraform を使ったデプロイ構成の作り方
• 生成 AI をアプリ組込時に、プロンプトだけでなくツール設計やデータの持たせ方が重要になること
• 図やドキュメントを整備しながら、第三者に伝わる形で成果物をまとめることの大切さ

特に、「AI に答えさせる」だけではなく「どのファイルを探し、どの情報を根拠に回答するか」を考えながらツールを設計した点が大きな学びでした。
